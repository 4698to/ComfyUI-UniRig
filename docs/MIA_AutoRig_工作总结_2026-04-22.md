# MIA AutoRig 修改工作总结（2026-04-22）

本文总结今天围绕 `MIA AutoRig` 的问题排查、功能调整与导出一致性修复。  
说明：本报告基于对话过程回顾整理，不依赖 git 状态对比。

## 一、背景与核心问题

今天的主要反馈集中在以下几类问题：

- `MIA` 导出的模型与输入模型存在明显尺度偏差（多次出现接近 100 倍问题）。
- 角色轴心与地面对齐不稳定，出现盆骨过低、落点异常等表现。
- 用户希望默认行为是“保持输入尺寸 + 脚底贴地”，减少隐式二次校正。
- 需要在导出前支持可选添加 `Root` 骨骼，且可控制开关。
- 需要更直观可见的关键日志，便于在 ComfyUI worker 控制台定位问题。

## 二、排查过程与结论

### 1) 尺度异常的根因判定

通过多轮日志与结果对照，确认问题并非单点，而是由以下因素叠加导致：

- 模板骨架（`assets/animation_characters/mixamo.fbx`）存在对象级缩放语义（常见 0.01）。
- 导出链中场景单位、FBX 单位参数、对象变换恢复之间存在耦合。
- 在不同阶段使用“绝对数值”或“错误参考轴”计算目标高度，会触发额外单位错配。

### 2) “注释掉核心仍 100 倍”现象的解释

该现象证明：即使移除部分保尺度逻辑，FBX 导出单位链路依然可单独造成 100 倍偏差。  
因此后续重点转向“导出单位与对象变换”的显式控制。

### 3) 关于 hips 目标参数的结论

用户指出“脚底贴地成立时，盆骨高度不应固定值硬设”，该判断合理。  
后续方案改为：

- 默认不做 hips 二次高度校正。
- 将“保持输入尺寸 + 脚底贴地”设为主路径。

## 三、今日已完成修改（按主题）

## A. `igl` 缺失与导入安全（早期修复）

- 在 `nodes/mesh_io.py` 中修复 `igl` 导入缺失时的报错链：
  - 先初始化 logger，再执行可选依赖导入。
  - `trimesh` 失败后 fallback 到 `igl` 之前先判空，避免 `None.read_triangle_mesh`。

## B. MIA 导出稳定性与可观察性

- 在 `nodes/mia_inference.py` 增加关键可见日志（`[MIA]` 前缀，stdout 可见）：
  - 运行选项
  - 输入高度快照
  - 保尺度校正因子
  - 导出前单位与 armature scale
  - 导出前 debug `.blend` 保存路径

- 新增导出前 `.blend` 调试保存：
  - 与输出 FBX 同名保存，便于复现定位。

## C. 默认行为收敛为“保尺寸 + 脚底贴地”

- 从参数层面移除了会引起歧义的高度校正入口：
  - 已去掉 `target_hips_height_m`
  - 已去掉 `normalize_hips_height`

- 保留并固定主路径：
  - 输入尺度快照（在推理早期获取）
  - 导出阶段按输入高度匹配
  - 脚底贴地处理

## D. Root 骨骼支持

- 在 `MIA` 导出链中增加了可选 `Root` 骨骼创建能力：
  - 新增节点参数：`add_root_bone`（默认 `True`）
  - 导出时可选择是否添加 `Root`
  - 添加时将其放置在原点，并挂接根级骨骼

## E. FBX 导出单位参数调整

- 导出参数改为显式设置（而非完全依赖默认值），并多次针对反馈微调。
- 当前策略已进入“显式单位 + 可见日志”的可诊断状态，便于继续收敛到目标 DCC（3ds Max）的一致表现。

## 四、当前版本与用户目标的对齐情况

已实现并确认的方向：

- 默认路径聚焦“保持输入模型尺寸 + 脚底贴地”。
- 可选添加 `Root` 骨骼，并可开关控制。
- 关键诊断信息在控制台可见，排查效率显著提高。

仍需持续关注的点（与 DCC 导入策略相关）：

- 3ds Max 侧对 FBX 单位与对象缩放值的解释，可能仍与 Blender 导出语义存在偏差。
- 需要继续基于控制台日志与导入结果做一轮参数定型（尤其是单位写出策略）。

## 五、经验沉淀

- 对跨 DCC 的 FBX 问题，优先做“可观测性”建设，再做参数收敛。
- 仅依赖“看起来高度一致”不够，还需同时观察导入后的对象缩放值。
- 模板骨架缩放、场景单位、导出参数三者必须统一管理，否则容易出现 100 倍级误差。

## 六、后续建议（下一步）

- 固化一套“面向 3ds Max”的导出参数预设（并在节点中可切换）。
- 增加导出后校验日志模板（尺寸、单位、骨架根层级、Root 是否存在）。
- 在文档中补充“Blender/Max 导入设置建议”，减少使用侧误差。
